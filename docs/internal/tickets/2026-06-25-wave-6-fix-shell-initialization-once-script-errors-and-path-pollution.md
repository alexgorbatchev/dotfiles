# Wave 6: Fix Shell Initialization Once-Script Errors and PATH/fpath Pollution

## Problem

The shell profile initializers generated by Go suffer from two critical correctness and integration bugs compared to the legacy TypeScript implementation:

1. **Once-Scripts Execution Error Pollution**: Go writes hardcoded absolute `source` (or `&` in PowerShell) statements for every once-script inside the main shell configuration file (`main.zsh`, `main.bash`, `main.ps1`):
   ```go
   // pkg/orchestrator/orchestrator.go
   scriptLines = append(scriptLines, fmt.Sprintf("source %q", onceFilePath))
   ```
   Under both TS and Go, the executed once-script includes code to delete itself after running (e.g., `rm -f "${(%):-%x}"`). 
   * **The Bug**: On the first shell startup, the script executes and successfully deletes itself. On every **subsequent** terminal startup, the shell profile still attempts to source the deleted file, causing noisy and annoying terminal initialization errors:
     * *Zsh*: `source: no such file or directory: .../once-001.zsh`
     * *Bash*: `bash: .../once-001.zsh: No such file or directory`
   TypeScript avoids this by generating a dynamic glob loop (e.g., using `for once_script in "${this.onceScriptDir}"/*.zsh(N); do [[ -f "$once_script" ]] && source "$once_script"; done`), where deleted files are skipped cleanly.

2. **Unconditional PATH/fpath Pollution**: Go unconditionally prepends directories to `PATH` and Zsh `fpath` in the generated `main` shell configurations.
   * **The Bug**: If a developer starts multiple nested subshells, or manually sources their shell config repeatedly, their `$PATH` and `$fpath` env variables will be flooded with redundant duplicate lines (e.g., `/home/user/bin:/home/user/bin:/usr/bin...`). TS handles this by generating conditional checks (such as `if [[ ":$PATH:" != *":${targetDir}:"* ]]`) and using Zsh's native array uniqueness flag (`typeset -U fpath`).

## Why this matters

A dotfiles manager must operate silently and robustly. Flooding the user's shell startup with "file not found" errors on every terminal opening degrades user experience. Flooding `$PATH` with hundreds of duplicates causes performance degradation during command resolution.

## Observed context

- Codebase files affected:
  - `pkg/orchestrator/orchestrator.go` (generates hardcoded once-script initializers and path modifications)
  - `pkg/shellinit/shellinit.go` (formats emissions)

## Desired outcome

The Go shell profile emission generator implements dynamic once-script matching loops and idempotent environment modifications, bringing complete behavioral parity and silence to the user's shell startup.

## Acceptance criteria

- [ ] Refactor Go's shell profile generator in `pkg/orchestrator/orchestrator.go` to stop writing hardcoded absolute paths for once-scripts into the generated main shell config file.
- [ ] Implement dynamic glob matching loops for each supported shell inside `pkg/shellinit/shellinit.go`:
  - **Zsh**: Use a dynamic glob matching loop with the `(N)` null-glob flag:
    ```zsh
    for once_script in "/path/to/.once"/*.zsh(N); do
      [[ -f "$once_script" ]] && source "$once_script"
    done
    ```
  - **Bash**: Use conditional matching with `nullglob` enabled inside a subshell:
    ```bash
    (shopt -s nullglob; for once_script in "/path/to/.once"/*.sh; do [[ -f "$once_script" ]] && source "$once_script"; done)
    ```
  - **PowerShell**: Loop through child items silently:
    ```powershell
    if (Test-Path "/path/to/.once") {
      Get-ChildItem -Path "/path/to/.once" -Filter "*.ps1" | ForEach-Object { & $_.FullName }
    }
    ```
- [ ] Add conditional presence checks to Go's generated `PATH` appends, preventing duplicates on repeated sourcing:
  - **POSIX (Zsh/Bash)**: Wrap the export in a substring comparison check:
    ```bash
    if [[ ":$PATH:" != *":/path/to/bin:"* ]]; then
      export PATH="/path/to/bin:$PATH"
    fi
    ```
  - **PowerShell**: Wrap in a `-notlike` conditional guard:
    ```powershell
    if ($env:PATH -notlike "*/path/to/bin*") { $env:PATH = "/path/to/bin;$env:PATH" }
    ```
- [ ] For Zsh, generate `typeset -U fpath` prior to appending custom completion directories to enforce native Zsh array uniqueness.
- [ ] Write unit tests inside `pkg/shellinit/shellinit_test.go` confirming that generated shell emissions contain the correct glob loop syntaxes and path protection guards.
